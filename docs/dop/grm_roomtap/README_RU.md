# GRM RoomTap — прослушка помещений

Отдельный аддон для Garry's Mod: чипы прослушки помещений, серверные стойки хранения и специальный компьютер мониторинга.

## Что делает

- **Чип прослушки** (`grm_roomtap_chip`) фиксирует в радиусе:
  - обычные текстовые реплики игроков;
  - вход игрока в зону, выход из зоны и отключение;
  - ID чипа, подпись, ручной сектор, автоматический сектор карты, радиус и сетевой канал.
- Чипы работают **сквозь стены** — по чистому радиусу, как было задано.
- Запись производится только тогда, когда существует включённая **серверная стойка** (`grm_roomtap_server`) с таким же каналом.
- Файлы хранятся на сервере в формате JSONL:
  
  ```text
  garrysmod/data/grm_roomtap/records/<карта>/<канал>/YYYY-MM-DD.jsonl
  ```

- **Компьютер мониторинга** (`grm_roomtap_terminal`) показывает последние записи, статус стоек и перечень активных чипов/секторов.
- Чипы не имеют 3D-подписей и индикаторов: это выбранный скрытый режим.
- Поддерживаются постоянная админская расстановка и временное оборудование игроков на 1 или 2 часа.

> Garry’s Mod не передаёт серверу исходный голосовой звук игроков. Поэтому аддон корректно реализует выбранный режим **журнала/расшифровки**: текст и события присутствия, без записи raw-аудио.

## Установка

1. Скопируйте папку **`grm_roomtap`** в:

   ```text
   garrysmod/addons/grm_roomtap/
   ```

2. Убедитесь, что итоговая структура имеет вид:

   ```text
   garrysmod/addons/grm_roomtap/lua/autorun/sh_grm_roomtap_config.lua
   garrysmod/addons/grm_roomtap/lua/autorun/server/sv_grm_roomtap.lua
   garrysmod/addons/grm_roomtap/lua/autorun/client/cl_grm_roomtap.lua
   ```

3. Перезапустите сервер или смените карту.

Зависимость от GRM Currency не обязательна. Если `GRM.HasMoney` и `GRM.TakeMoney` существуют, магазин использует GRM; без экономики установка временных устройств будет бесплатной.

## Используемые модели

| Оборудование | Модель |
|---|---|
| Чип прослушки | `models/jaanus/wiretool/wiretool_controlchip.mdl` |
| Серверная стойка | `models/props_silo/silo_server_d.mdl` |
| Компьютер | `models/props/cs_office/computer.mdl` |

Модели и все основные числа настраиваются в:

```text
lua/autorun/sh_grm_roomtap_config.lua
```

## Сети, каналы и сектора

У каждого чипа и стойки есть поле **«Канал сети»**.

Пример:

- чип «Допросная №2» → канал `police_01`;
- серверная стойка «Архив полиции» → канал `police_01`;
- терминал → отображает журнал всех каналов для игроков с правом доступа.

Если нет включённой стойки нужного канала, чип не создаёт записи.

У чипа доступны:

- **подпись** — понятное название устройства;
- **ручной сектор** — например, `Кабинет 101`;
- **автосектор** — рассчитывается по координатам (`X... / Y...`), размер ячейки задаётся `GridSectorSize`.

## Права доступа

Система независима от `/phone_access`.

Superadmin открывает меню:

```text
/roomtap_access
!roomtap_access
roomtap_access
```

Там выдаётся доступ целой фракции, отдельным ролям или отделам из глобальной таблицы `Factions`.

Игроки с доступом могут:

- настраивать чипы и серверные стойки;
- использовать терминал и читать записи всех каналов;
- покупать временное оборудование, если `ShopRequireEquipmentAccess = true`.

---

## Временное оборудование игроков

Команды:

```text
/roomtapshop
!roomtapshop
roomtap_shop
```

Магазин выдаёт оборудование на выбранный срок:

- 1, 2, 3, 5 или 10 часов;
- 24 часа;
- 2, 3 или 5 суток;
- 1 неделю.

Доступны все типы оборудования:

- чип;
- серверная стойка;
- компьютер.

Базовая цена каждого предмета указана за 1 час. Множители сроков настраиваются в `TemporaryDurations` файла `sh_grm_roomtap_config.lua`.

Временные предметы сохраняются в:

```text
data/grm_roomtap/temporary_equipment.json
```

и переживают рестарт сервера, пока не закончится их срок. Убрать ближайший свой временный предмет:

```text
/roomtap_remove
!roomtap_remove
roomtap_remove
```

Владелец временного устройства может нажать `E` по нему и выбрать **«Запросить постоянное сохранение»**. Superadmin видит такие заявки:

```text
/roomtap_requests
!roomtap_requests
roomtap_requests
```

После одобрения предмет становится постоянным оборудованием карты.

## Постоянная админская расстановка

Entity доступны администраторам в категории **GRM RoomTap**. После установки:

1. Нажмите `E` по чипу или стойке и настройте их.
2. Superadmin нажимает **«Сохранить навсегда»**.
3. Либо сохраните все отмеченные предметы командой:

   ```text
   grm_roomtap_save
   ```

Загрузка постоянной расстановки:

```text
grm_roomtap_load
```

Файл карты:

```text
data/grm_roomtap/maps/<имя_карты>.json
```

## Служебные команды

| Команда | Назначение |
|---|---|
| `grm_roomtap_save` | Сохранить постоянное оборудование карты (superadmin) |
| `grm_roomtap_load` | Загрузить постоянное оборудование карты (superadmin) |
| `grm_roomtap_reload` | Перечитать доступ и последние записи (superadmin) |
| `grm_roomtap_debug` | Вывести число устройств, записей и временных объектов |
| `roomtap_access` | Меню прав (superadmin) |
| `roomtap_requests` | Одобрение запросов на постоянное сохранение (superadmin) |

## Хук интеграции

После успешной записи вызывается серверный хук:

```lua
hook.Add("GRM_RoomTapRecorded", "MyIntegration", function(record, chip, storage)
    -- record: таблица записи
    -- chip: entity grm_roomtap_chip
    -- storage: entity grm_roomtap_server
end)
```
