# GRM Vehicle Keys — исправленный комплект

## Установка

Скопируйте папку `grm_vehicle_keys_fixed` в `garrysmod/addons/`.

Перед установкой удалите или отключите старые версии файлов VK, особенно старый `vehicle_keys_swep.lua`. Одновременно должен работать **только один** комплект системы, иначе старый SWEP снова может перезаписать защищённые обработчики net-сообщений.

Структура:

```text
lua/autorun/sh_vehicle_keys.lua
lua/autorun/server/sv_vehicle_keys.lua
lua/autorun/client/cl_vehicle_hud.lua
lua/weapons/vehicle_keys_swep.lua
```

## Исправления

- Только `sv_vehicle_keys.lua` регистрирует `VK_Keys_Give` и `VK_Keys_Revoke`.
- Сервер проверяет, что владелец/superadmin смотрит на нужную машину в допустимом радиусе.
- Новый владелец автоматически получает ключ и SWEP.
- Члены фракции с фракционным транспортом автоматически получают SWEP.
- Сброс владельца разблокирует транспорт.
- Состояние владельца и замка синхронизируется через NW2 и net.
- В меню ключей виден настоящий статус ключа игрока: «Выдать» либо «Отозвать».
- Поддержаны стандартные транспортные entity, Simfphys, распространённые LVS/LFS классы и Vehicle Dealer-маркер `VD_ID`.
- Фракции ищутся в глобальной таблице `Factions` по SteamID и SteamID64.

## Команды

### Владелец личной машины

```text
vk_givekey <точный ник или SteamID>
vk_revokekey <точный ник или SteamID>
```

Нужно смотреть на свою машину с близкого расстояния.

### Superadmin

```text
vk_setowner <точный ник или SteamID>
vk_setfaction <полное название фракции>
vk_clearowner
```

Нужно смотреть на нужный транспорт.

## Управление SWEP

```text
ЛКМ — заблокировать / разблокировать
ПКМ — открыть / закрыть двери
R   — меню выдачи и отзыва личных ключей
```

Меню по `R` работает только у личного транспорта игрока или у superadmin. Фракционные машины открываются всеми членами фракции, но не имеют персонального меню ключей.

## Важно о дверях

Разные сборки Simfphys, LVS и LFS используют разные API дверей. Комплект безопасно пробует распространённые методы (`ToggleDoors`, `SetDoorsOpen`, `OpenDoors`, `CloseDoors` и двери из entity table). Если конкретная модель не поддерживает их, замок и доступ продолжат работать, но ПКМ может сообщить, что её двери не поддерживаются.
